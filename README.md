### 冬季战场v1.0.6 alpha_test测试报告：
2026/2/7

使用i7-14650HX 16G Laptop平台作为服务器，服务器兼作为测试员CE_PT运行Minecraft客户端的设备
服务器最低TPS约为10（掉刻）
极限情况下，有6人参与战斗，另有1人在空中观战

根据大家的反馈，我们整理出以下观点：
大战场玩法本身相当有趣，但是——
数值平衡有待改进
神枪·冈格尼尔左键突进冷却时间太短，很难正常普攻
狙击手太强势了，玩家很容易死掉
虽然掉刻不是很影响游玩，网络卡顿较为严重
有一堆BUG！！！
作为离线服务器，无法正确显示其他玩家的皮肤，也没有加装相应的补丁解决这个问题
赐福系统不平衡，如果不使用**存护**赐福，玩家很容易死掉
……

**在此之外，还有如下问题：**

极少有人使用步话机系统，且QQ语音通话较为混乱，难以有效组织
（有玩家指出可以使用服务端通话插件等方式解决这个问题，我们正在考虑）

没有使用replay mod记录游戏进程
（很难反复检查测试过程，只能靠回忆和日志）

新手引导几乎完全缺失，新人难上手
（比如在寻找加入阵营的按钮和分辨自己所处阵营时比较混乱。因为冬战成员用书因为技术问题没有迁移，这个问题更加突出了。
我们正在制作这样一套系统：
准备系统、阵营选择、兵种选择、赐福选择、出生点选择从地图内集成到对话框中，并使用**成书**作为触发开关，做成新一代的**冬战成员用书**）

存在玩家们把掉刻误认为是网络问题的可能性
（因为只有常驻的几位测试员通过mod发现了服务器掉刻）

中途加入的玩家会加载出生点区块以及它旁边的红石机器
（可能是掉刻的原因之一，我们正在加急制作独立于原有存档的冬战地图，并禁用生物生成等一切无关功能）

有几位中途加入的玩家无视语音沟通，以创造模式误入战场
（我们想到的办法是，将默认游戏模式调节为旁观模式，如果仍有这样的情况，让管理员手动调整他们的游戏模式为旁观模式）

本次测试中，各兵种出场频率如下：
第一梯队：近战兵 狙击手
第二梯队：冈格尼尔干员 
第三梯队：普通兵 三叉戟兵 立体机动装置 板载兵
第四梯队：游骑兵（没有人玩） 

最后，感谢大家参与我们的测试，如果有其他的想法，欢迎与我们联系，这对我们游戏的改进非常重要，大家的支持是我们最好的动力！

### 传送系统更新
传送系统更新到**第三版**
传送系统从**基于命令方块+压力板**升级为**基于对话框**，在**冬战成员用书**的主界面即可跳转

我们保留了第二版传送器，但推荐大家使用第三版传送器
因为传送位置增加了**条件判断**，所以拆除了原本**按条件限制传送**的隔间，并移除了相关代码
（因为新代码支持联动到**游戏流程计算器（冬战数据包搭建的检测战斗阶段的计分板）**检测每一个启动器的开启情况，第三版传送系统将传送逻辑从旧版的**占领战区才能解锁战区和启动器**改为**激活启动器后就能传送到该启动器**，再加上其他的检测代码，所以不再需要物理墙来阻隔）
将**第二版传送系统**的命令方块使用的代码从命令**升级为**第三版传送系统**所调用的**函数**
